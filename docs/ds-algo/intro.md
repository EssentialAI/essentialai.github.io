## What are data structures?
Data Structures are different ways of organizing data on your computer, that can be used efficiently. For example, we have a bunch of woods that are not organized. If we want to search a black colored woodden plank, we need to search between all the planks available (in the worst case scenario). Alternatively, in an organized case it becomes very easy. When it comes to data structures, we do a similar thing, organizing the data.

From a software application point of view, the performance of your application depends on how the data is stored, organized, and grouped together during program execution. Example is queue data structure for ticket purchasing. A programmer must know what data structure to use in a given scenario.

## What is an Algorithm?
One definition to algorithm can be <span class = 'italic'>A set of steps to accomplish a task</span>. In the previous example containing the woodden planks, we have to accomplish several tasks including choosing which color to select for building, then flooring, and building and so on. For example, companies like Facebook use Compression algorithms, to tranmit live feed across the internet. Companies like Google use Dijkstra's Algorithm to find the shortest path between two points.

## Why Data Structures and Algorithms are important?
Programmers take data as input, perform data processing and provide user with processed data. DS Algos are essential during this data processing. For example, in the case of a library, each book is data, arranging them in racks represents Data Structures, and the process of finding a book represents an Algorithm.

By asking questions related to DS Algos, the interviewer checks the problem solving abilities of the candidate. Additionally, interviewer has limited time to evaluate a candidate, this means that if a candidate is proficient in DS Algos, he/she can learn a new technology or language very easy because they just need to learn the syntax.

## Types of Data Structues

1. Primitive Data Structures

The DS that are pre-built in the programming language itself and can be used by the programmer are called Primitive.
Examples:
* Integer
* Float
* Character
* String
* Boolean
