Gradient Descent

In the previous articles that deal with 1-D and multiple Linear Regression, we had to minimize the loss function (EEE) (parametrized by weights www). The aim is to find the optimal weight vector www where EEE is minimum.

To find the minimum value of EEE, we discussed a naive approach that uses calculus. However, we observed several caveats of the naive approach such as:

  1. The equations for partial derivatives become complex very easily (even for the Linear Regression problem).
  2. There can be instances where the inverse of XTXX^TXXTX does not exist and naive approach cannot be used.
  3. We would also like an iterative solution that does not become computationally intensive.
Gradient Descent offers a generalizable solution to minimize the loss function and is used everywhere throughout Machine Learning.

The main idea here is that, we take tiny steps in the direction of gradient of EEE until we reach the minimum value of EEE.

Gradient Descent for Linear Regression​

Let's first restate the Linear Regression problem and understand how Gradient Descent can be useful.

  1. We have a bunch of data points (X1,Y1),(X2,Y2),...,(Xn,Yn)(X_1, Y_1), (X_2, Y_2),...,(X_n, Y_n)(X1​,Y1​),(X2​,Y2​),...,(Xn​,Yn​).
  2. We need to find the slope (mmm) and intercept (bbb) of the line yyy that best explains the relation between XXX and YYY. (1-D Linear Regression).
  3. In case of multiple regression, we need to find the vector wD×1w_{D \times 1}wD×1​, that best explains the relationship between XN×DX_{N \times D}XN×D​ and y⃗N×1\vec{y}_{N \times 1}y​N×1​ such that,
y⃗N×1=XN×DwD×1\vec{y}_{N\times1} = X^{}_{N\times D}w^{}_{D\times1}y​N×1​=XN×D​wD×1​

  • NNN →\rightarrow→ Number of data samples
  • DDD →\rightarrow→ Number of features in XXX that are used to find www (for 2-D Linear Regression D = 2, 1-D, D=1)
E=1N∑i=1N(yi−yi^)2=1N∑i=1N(yi−wTxi)2\begin{align} E = \frac{1}{N}\sum_{i=1}^{N}(y_{i}-\hat{y_{i}})^2 = \frac{1}{N}\sum_{i=1}^{N}(y_{i}-w^Tx_{i})^2 \end{align}E=N1​i=1∑N​(yi​−yi​^​)2=N1​i=1∑N​(yi​−wTxi​)2​​
∂E∂wj=2N∑i=1N(yi−wTxi)(−∂(wTxi)∂wj)\frac{\partial E}{\partial w_{j}} = \frac{2}{N}\sum_{i=1}^{N}(y_{i}-w^Tx_{i})(-\frac{\partial (w^Tx_{i})}{\partial w_{j}})\\∂wj​∂E​=N2​i=1∑N​(yi​−wTxi​)(−∂wj​∂(wTxi​)​)
=2N∑i=1N(yi−wTxi)(−xij)= \frac{2}{N}\sum_{i=1}^{N}(y_{i}-w^Tx_{i})(-x_{ij})=N2​i=1∑N​(yi​−wTxi​)(−xij​)

The above equation can be rewritten in matrix form as:

∂E∂w≈XT(Yhat−Y)\begin{align} \frac{\partial E}{\partial w} \approx X^T(Y_{\text{hat}}-Y) \end{align}∂w∂E​≈XT(Yhat​−Y)​​
Yhat=y⃗N×1Y_{\text{hat}} = \vec{y}_{N \times 1}Yhat​=y​N×1​

(2)(2)(2) represents gradient of error function EEE. How do we take tiny steps in the direction of the gradient?

To achieve this, we use time-steps (also called as epochs). At every time-step t+1t+1t+1, we use the values of previous gradients wtw_{t}wt​ to update the gradients wt+1w_{t+1}wt+1​. We use the learning rate (α\alphaα) to take small steps.

wt+1=wt−αN(XT(Yhat−Y))\begin{align} w_{t+1} = w_t-\frac{\alpha}{N} (X^T(Y_{\text{hat}}-Y)) \end{align}wt+1​=wt​−Nα​(XT(Yhat​−Y))​​

Usually the value of learning rate is in the range of 0.01 to 0.05. This is a hyper-parameter that is to be optimized.

  1. Initialize random weights wD×1w_{D \times 1}wD×1​.
  2. Find YhatY_{\text{hat}}Yhat​ using Y⃗N×1=XN×DwD×1\vec{Y}_{N\times1} = X^{}_{N\times D}w^{}_{D\times1}YN×1​=XN×D​wD×1​
  3. Iteratively update www using wt+1=wt−αN(XT(Yhat−Y))w_{t+1} = w_t-\frac{\alpha}{N} (X^T(Y_{\text{hat}}-Y))wt+1​=wt​−Nα​(XT(Yhat​−Y))
  4. As YhatY_{\text{hat}}Yhat​ get closer and closer to YYY, the weights www converge to optimal weights.

A small visualization of loss minimization​

What happens when loss reaches zero?​

Loss EEE = 0 represents, Yhat−Y=0Y_{\text{hat}} - Y = 0Yhat​−Y=0. In this case, the weight wt+1w_{t+1}wt+1​ becomes wtw_twt​ (as the term on the right side vanishes). This means, the weights converge to wtw_twt​ and the model does not learn anymore. Thus the optimal weights are preserved.

Summary​

This article clearly explains Gradient Descent along with the underlying mathematics. While providing the benefits listed in the start of this article, Gradient Descent is also included in all the popularly used frameworks such as Tensorflow and Theano. This enables us to focus more on hyperparameter tuning rather than getting involved in complex mathematics.

Code​

import numpy as np
import matplotlib.pyplot as plt
from sklearn import datasets
import pandas as pd
from sklearn import preprocessing
from sklearn.linear_model import LinearRegression
import warnings
warnings.filterwarnings("ignore")

def read_boston_data():
    boston = datasets.load_boston()
    df = pd.DataFrame(data = boston.data)
    df.columns = boston.feature_names
    df['Target'] = boston.target
    df= df.rename(columns = {'Target':'Price'})
    # Taking RM and LSTAT as features as they are highly correlated to price.
    x_RM = preprocessing.scale(df['RM'])
    x_LSTAT = preprocessing.scale(df['LSTAT'])
    y = preprocessing.scale(df['Price'])
    
    plt.scatter(y, x_RM, s=5, label = 'RM')
    plt.scatter(y, x_LSTAT, s=5, label = 'LSTAT')
    plt.legend(fontsize=15)
    plt.xlabel('Average number of rooms & Low status population', fontsize=15)
    plt.ylabel('Price', fontsize=15)
    plt.legend()
    plt.show()
    
    return x_RM, x_LSTAT, y
    
x1,x2,y = read_boston_data()
x = np.c_[np.ones(x1.shape[0]),x1, x2]
# Parameters required for Gradient Descent
alpha = 0.0001   #learning rate
m = y.size  #no. of samples
np.random.seed(10)
theta = np.random.rand(x.shape[1])

def gradient_descent(x, y, m, theta, alpha):
    cost_list = []   #to record all cost values to this list
    theta_list = []  #to record all theta_0 and theta_1 values to this list 
    prediction_list = []
    run = True
    cost_list.append(1e10)    #we append some large value to the cost list
    i=0
    while run:
        prediction = np.dot(x, theta)   #predicted y values theta_0*x0+theta_1*x1
        prediction_list.append(prediction)
        error = prediction - y
        cost = 1/(m) * np.dot(error.T, error) #  (1/2m)*sum[(error)^2]
        cost_list.append(cost)
        theta = theta - (alpha * (1/m) * np.dot(x.T, error))   # alpha * (1/m) * sum[error*x]
        theta_list.append(theta)
        if cost_list[i]-cost_list[i+1] < 1e-9:   #checking if the change in cost function is less than 10^(-9)
            run = False

        i+=1
    cost_list.pop(0)   # Remove the large number we added in the begining 
    return prediction_list, cost_list, theta_list

prediction_list, cost_list, theta_list = gradient_descent(x, y, m, theta, alpha)
theta = theta_list[-1]

print('Model-Intercept : {}'.format(round(theta[0],3)))
print('Model-Theta_0 : {}'.format(round(theta[1],4)))
print('Model-Theta_1 : {}'.format(round(theta[2],4)))

plt.title('Cost Function J', size = 30)
plt.xlabel('No. of iterations', size=20)
plt.ylabel('Cost', size=20)
plt.plot(cost_list)
plt.show()

ys = y
xs = np.c_[x1,x2]

xs = preprocessing.scale(xs)
ys = preprocessing.scale(ys)

lm = LinearRegression()

#Fitting the model
lm = lm.fit(xs,ys)
intercept = lm.intercept_
Theta_0 = lm.coef_[0]
Theta_1 = lm.coef_[1]

print('Sklearn-Intercept : {}'.format(round(intercept,3)))
print('Sklearn-Theta_0 : {}'.format(round(Theta_0,4)))
print('Sklearn-Theta_1 : {}'.format(round(Theta_1,4)))

Output​
